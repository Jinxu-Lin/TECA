# Innovator Debate — TECA

## 五项立项判断

### 1. 最强继续理由
TDA 领域确实缺少一个不依赖重训练的外部验证通道，这是一个真实且未解决的方法论缺口。如果 TECS 能提供哪怕是弱但稳定的区分信号，它会成为 TDA 评估的标准补充工具。更重要的是，即使 TECS 全面失败（所有方法 cosine 都接近零），这本身就是关于"知识是否以局部化方式存储"的强实验证据，具有独立发表价值。

### 2. 最危险失败点
两端同时不可靠导致结果不可解释。IF 在 LLM 上有参数-行为断层（2409.19998），ROME 的 causal tracing 定位有争议且 side effects 广泛。如果 TECS 计算出来是 0.05，你无法区分：(a) TDA 方法差，(b) ROME 定位不准，(c) 知识本身是分布式的，(d) 余弦相似度不是正确度量。四路混淆使得任何结果都难以下结论。

### 3. 被施压的核心假设
**假设 1**（TDA 梯度方向与 ROME 编辑方向指向相同子空间）最脆弱。这是整个工作的地基，但没有任何先验理论或实验支撑。TDA 的梯度是关于 loss 对参数的导数（聚合了所有 token 位置），ROME 的编辑向量是通过 causal tracing + constrained least squares 构造的 rank-1 update。这两个量在数学构造上没有理由对齐——一个是优化方向，一个是因果干预方向。

### 4. 关键证据要求
在提交完整项目之前，需要一个 5 小时以内可完成的探针实验：在 GPT-2 级别模型上，对 5-10 个已知事实三元组，分别计算 ROME 编辑向量和 IF 梯度向量，检查余弦相似度的分布。如果分布均值显著高于随机基线（随机层的余弦），则假设 1 获得初步支撑。如果与随机无异，项目应立即终止。

### 5. 本视角建议
**Hold** — 核心假设过于脆弱，但方向的"失败也有价值"特性使其不应直接 Stop。需要探针实验提供 go/no-go 信号。

## 天花板评估

**当前设计的最乐观贡献**：中等。即使一切顺利（TECS 有效区分好坏 TDA 方法），这仍然是一个评估指标论文，贡献天花板是 poster level。原因：

1. 它不改进任何 TDA 方法本身，只提供一个新的评估维度
2. 指标的可信度取决于 ROME 的可靠性，而 ROME 本身有争议，这限制了社区接受度
3. 适用范围窄——仅适用于可以做知识编辑的事实性知识场景，无法推广到一般的 TDA 评估

**理论天花板**：如果发现 TECS 揭示了"知识存储的几何结构"（如某些层的子空间确实同时被编辑和归因捕捉），这可以升级到 spotlight，但需要远超当前设计的分析深度。

## 升维角度

### 升维 1：从"评估指标"升级为"参数空间知识探测框架"

当前设计把自己定位为"TDA 的评估工具"，这是自我矮化。更大的故事是：**TECA 可以成为探测 LLM 内部知识表征几何结构的通用框架**。

具体做法：不只比较 TDA vs ROME，而是构建一个"参数空间知识探针矩阵"——横轴是多种知识定位方法（ROME、MEMIT、causal tracing、activation patching、probing classifiers 的梯度），纵轴是多种归因方法（IF、TRAK、datamodels、gradient similarity）。TECS 变成矩阵中的一个 cell，整个矩阵揭示的是"不同方法看到的知识几何是否一致"。

这将贡献从"一个指标"升级为"一个关于知识表征的实证发现"。如果矩阵显示某些方法簇高度一致而其他方法孤立，这本身就是关于 LLM 内部结构的重要发现。

### 升维 2：从静态比较到动态一致性追踪

当前设计是在训练完成后做一次性比较。更有野心的做法：**追踪训练过程中 TDA 方向和编辑方向的动态对齐变化**。

在模型训练的不同 checkpoint，分别计算 TECS。如果 TECS 在训练早期为零、随着知识巩固逐渐上升，这就提供了"知识何时从分布式变为局部化"的实验证据——直接回应 knowledge localization 的争论。这会把论文从 TDA 评估拉到 mechanistic interpretability 的核心议题。

## 保守之处

1. **度量选择过于简单**：余弦相似度假设方向对齐是线性的。知识可能存储在非线性子流形上，此时余弦相似度会系统性地低估真实对齐程度。应至少探索 CKA（Centered Kernel Alignment）或子空间投影重叠度（principal angle）作为替代度量。

2. **单层比较是自我设限**：仅在 ROME 编辑层 l* 比较是在赌 knowledge locality 假说成立。如果知识是跨层分布的（越来越多证据支持这一点），单层 TECS 会给出误导性的低分。应从一开始就设计跨层的 TECS profile，而非事后补充。

3. **没有利用编辑的因果结构**：ROME 不只提供方向，还提供编辑的因果效果（编辑前后的输出变化）。当前设计丢弃了这个信息。应该将 TECS 与编辑成功率联合分析——高 TECS 且编辑成功 vs 高 TECS 但编辑失败，这些案例的差异会提供关于 TECS 可靠性的关键诊断信息。

4. **回避了 scaling 问题**：在 GPT-2 上验证的 TECS 可能在 7B+ 模型上完全失效（IF 近似质量急剧下降）。当前设计没有 scaling 策略，这限制了实际影响力。

## 实现预判

- **探针实验**（GPT-2, 5-10 事实）：1-2 天，这是 go/no-go gate
- **完整 TECS 实现**（多 TDA 方法 x 多事实 x 单模型）：1-2 周
- **升维到知识探针矩阵**：额外 2-3 周，但影响力倍增
- **主要工程风险**：IF 在 LLM 上的计算成本和近似质量；ROME 实现的正确性验证
- **计算资源**：GPT-2 级别单卡可做；7B 需要多卡或显存优化
