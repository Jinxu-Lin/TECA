# Contrarian Debate — TECA

## 五项摘要

### 1. 最强继续理由

TDA 领域确实缺乏独立于重训练的评估通道。如果 TECS 能工作，它填补的是一个真实空白。这个问题本身是合法的。

### 2. 最危险失败点

**循环论证陷阱**：TECS 用两个各自不可靠的工具（TDA 和 ROME）互相验证。如果 ROME 的 causal tracing 不反映真实知识定位（已有争议），而 IF 的 iHVP 近似使其退化为梯度点积（2409.19998 已证实），那么 TECS 测量的是"两个 artifact 的巧合对齐"，而非"两个工具在真实知识位置上的一致性"。一致高不能证明两者都对，一致低也不能证明两者都错。TECS 在认识论上是悬空的。

### 3. 被施压的核心假设

**假设 1（TDA 方向与 ROME 方向对齐）** 和 **假设 2（ROME 反映知识位置）** 是致命的。

- **假设 1 — 支撑：弱**。这是整篇论文的立论基础，但没有任何先验理论保证对齐。TDA 计算的是"删除该样本对 loss 的影响方向"，ROME 计算的是"改变该事实输出所需的最小参数扰动"。这两个操作在数学上没有必然联系——前者是关于 loss landscape 的局部线性近似，后者是关于输出空间的 rank-1 约束优化。"在同一参数子空间操作"是一个修辞性类比，不是定理。
- **假设 2 — 被严重质疑**。Hase et al. 质疑 causal tracing 的解释力。ROME side effects（编辑一个事实影响其他事实）直接说明编辑不是精确的"知识定位"，而是在纠缠的参数空间中做粗暴的 rank-1 手术。如果 ROME 定位的是 artifact 而非知识，TECS 测量的就是"TDA 是否也定位到同一个 artifact"。

### 4. 关键证据要求

在投入完整实验之前，必须回答：

1. **Sanity check for circularity**：设计一个实验，能区分"TECS 高因为两者都定位到真实知识"和"TECS 高因为两者都被同一个 confound 驱动（如梯度范数大的层天然高 cosine）"。如果设计不出这个实验，整个项目在认识论上就是不可证伪的。
2. **ROME reliability quantification**：在 CounterFact 500 个事实上，ROME 编辑的 side effect 率和严重度是多少？如果 >30% 的编辑破坏其他事实，ROME 作为"知识定位"信号的可信度就不够。
3. **2409.19998 的 IF 退化问题**：iHVP 近似使 IF 退化为梯度点积。如果 TECS 用的 IF 实际上是梯度点积，那"IF TECS 高"只说明"梯度方向和编辑方向对齐"——这可能是平凡的（大梯度层 = 编辑层，因为两者都偏好高信息层）。

### 5. 本视角建议

**Hold**

## 完整论证

### I. 核心假设审查

#### 假设 1：TDA 方向与 ROME 方向对齐（支撑：弱）— 致命

这是项目的唯一赌注，但它不是从理论推导出来的，而是一个直觉类比。

**反事实场景**：TDA 梯度 $g_M$ 反映的是"删除训练样本后 loss 变化的方向"，这取决于整个 loss landscape 的曲率（Hessian）。ROME 的 $\Delta\theta$ 反映的是"在 rank-1 约束下，改变特定输出所需的最小参数变化"，这取决于输入-输出映射的局部 Jacobian。这两个方向没有理论上必须对齐的理由。它们可能碰巧在某些层对齐（因为高信息层同时是梯度大的层和编辑有效的层），但这种"对齐"是 confounded by layer importance，不是两者在语义上指向同一个知识。

**最坏情况**：TECS 对所有梯度方法都 > 0，但原因是"梯度范数大的层恰好也是编辑层"这一平凡关联。论文发表后被一行实验推翻：控制层级梯度范数后 TECS 消失。

#### 假设 2：ROME 反映知识位置（支撑：中）— 高危

ROME 的 causal tracing 声称知识定位在中间层 MLP。但：
- ROME side effects 意味着编辑不是精确定位，而是在纠缠空间中做手术
- Hase et al. 的质疑表明 causal tracing 可能测量的是"处理瓶颈"而非"存储位置"
- 如果知识是分布式存储的（这在神经科学和 DL 理论中都是主流观点），那么"知识在某一层"本身就是错误的前提

**反事实场景**：ROME 之所以"工作"（编辑成功），不是因为它找到了知识存储的位置，而是因为它找到了一个信息传播的瓶颈——在这里做 rank-1 扰动恰好能改变输出。这与 TDA 测量的"训练影响"是完全不同的概念。

#### 假设 3：IF/TRAK 归因可靠（支撑：中）— 高危

2409.19998 明确表明 iHVP 近似使 IF 退化为梯度点积。2602.09987 (Infusion) 表明 IF 在语言模型上的逆向优化几乎完全失败（0.1% rank flip）。如果 TDA 这一端都不可靠，"两端互验"就变成了"一端不靠谱去验证另一端不靠谱"。

#### 假设 4 和 5 是次要的，前三条不解决，后面无意义。

### II. 循环论证的不可逃避性

这是本项目最根本的认识论问题。

TECS 的声称是："如果 TDA 和 ROME 方向一致，说明两者都定位到了真实知识位置。"但这个推理需要一个前提："至少有一个是可靠的。"现有证据不支持这个前提——ROME 的可靠性有争议，IF 的可靠性有争议。

**两种结果都不可解释**：
- TECS 高 → 可能是"都定位到真实知识"，也可能是"都被同一个 confound 驱动"
- TECS 低 → 可能是"知识是分布式的"，也可能是"TDA 不行"，也可能是"ROME 不行"

没有外部锚点来区分这些解释。论文声称 TECS 是"独立验证通道"，但它实际上只是"第二个未经验证的通道"。

### III. 被低估的替代方案 / 竞争威胁

1. **直接改进 LDS**：与其发明新指标，不如解决 LDS 的已知问题（信噪比低 → 更多重训练 + 统计检验；随机性 → ensemble）。这更无聊但更可靠。
2. **Behavioral probing**：不在参数空间比较，而在行为空间比较——"ROME 编辑后模型行为变化"与"TDA top-k 样本删除后模型行为变化"的 correlation。这绕开了参数空间对齐的假设。
3. **2409.19998 的方向**：直接研究"参数空间距离与行为变化的关系"，这是一个更 fundamental 的问题，TECA 只是其表面应用。

### IV. 预期结果表的问题

预期 IF (精确 Hessian) TECS 高——但谁能在 GPT-2-XL（1.5B 参数）上计算精确 Hessian？实际用的是 EK-FAC 近似，而近似质量正是 2409.19998 质疑的。如果近似 IF ≈ 梯度点积，那"IF TECS 高"和"LoGra TECS 高"测量的是同一个东西，整个梯度方法阵列退化为一个点。

### V. 生死线

| 检查点 | 条件 | 若不满足 |
|--------|------|---------|
| Phase 1 第 1 周 | Random baseline TECS 的置信区间不包含任何梯度方法的 TECS | 指标无区分力，Stop |
| Phase 1 第 1 周 | 控制层级梯度范数后，梯度方法 TECS 仍显著 > 0 | 对齐是平凡 confound，Stop |
| Phase 1 第 2 周 | RepSim TECS 显著低于梯度方法 | 否则参数空间 vs 表示空间的区分不成立，Stop |
| 开工前 | 能设计出区分"真实对齐"和"confounded 对齐"的实验 | 不可证伪，Hold |

### VI. 总结判断

TECA 的核心 idea 有直觉吸引力，但在认识论上有根本性缺陷：用两个各自不可靠的工具互相验证，且没有外部锚点来打破循环。最强版本的 TECA 需要先解决循环论证问题（提供独立于 ROME 和 TDA 的 ground truth），但如果有这样的 ground truth，TECA 本身就不需要了。

**建议：Hold**，除非团队能在一周内设计出打破循环论证的实验方案。具体来说：找到一种方法证明"TECS 高 → 两者都对"而非"TECS 高 → 两者被同一 confound 驱动"。如果找不到，这个项目在科学上是不可证伪的，不值得投入。
